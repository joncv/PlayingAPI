Jquery.PlayingAPI v 1.0

介绍

Jquery.PlayingAPI是一个轻量级调试api接口的jquery插件.提供方法定义,加载,显示接口对象内容,调试结果,支持异步post,多文件异步post.

下载

引用

    
    <!-- 如果您已经在项目中使用了bootstrap2的主要样式,可以不引用jquery.playingapi-1.0.css -->
    <link href="jquery.playingapi-1.0.css" rel="stylesheet">
    <script src="jquery-1.8.3.min.js"></script>
    <script src="jquery.playingapi-1.0.js"></script>
    <!-- 如果您需要多文件post请引用jquery.form.js,建议引用 -->
    <script src="jquery.form.js"></script>

你可以参考以下代码去初始化您的playingapi.


<div id="mydiv"></div>
<script type='text/javascript'>
    $(document).ready(function () {
        var apiList = {
            list: [
                {
                    url: 'myurlwill404',
                    urlParams: 'myurlwill404/{testid}',
                    requestType: 'POST',
                    requiredName: 'name,password,sex,mobilephone',
                    name: 'name,password,sex,mobilephone',
                    value: 'show对象1(有文件上传),asdfasdf,男,13800138000',
                    urlDesc: '{testid:测试id}',
                    desc: '名字,密码,性别,手机',
                    uploadfile: {
                        name:'A文件,B文件,C文件',
                        value:'AFile,BFile,CFile'
                    }
                },
                {
                    url: 'myurlwill404',
                    urlParams: 'myurlwill404/{testid}',
                    requestType: 'POST',
                    requiredName: 'name,password,sex,mobilephone',
                    name: 'name',
                    value: 'show对象2,asdfasdf,男,13800138000',
                    urlDesc: '{testid:测试id}',
                    desc: '名字,密码,性别,手机',
                    uploadfile: {
                        name:'A文件,B文件,C文件',
                        value:'AFile,BFile,CFile'
                    }
                }
            };
    $("#mydiv").playingapi(apiList); 
</script>

当前playingapi的布局只是demo,实例中请根据实际情况调整.

show 方法

显示所定义的playingapi对象.


<script type='text/javascript'>
    $("#mydiv").playingapi("show", 1);
</script>

方法

名字 参数 返回 描述
show number 根据number的索引，显示内容.
playingapi 包含有list数组的任何对象 构造playingapi

数据

如果不是异步多文件post,请获取"papidata"参数,它是一个json.

文档

选项

属性 类型 默认 描述
list Array playingapi会根据它来渲染,它接受以下多个对象 
list:[
        {
            url: '',
            urlParams: '',
            requestType: '',
            requiredName: '',
            name: '',
            value: '',
            urlDesc: '',
            desc: '',
            uploadfile: {
                name: '',
                value: ''
            }
        },
        ...
    ]
list.url string 请求地址,当点击请求按钮时会根据该参数请求
list.urlParams string 地址参数(文本说明,不影响请求)
list.urlDesc string 地址描述(文本说明,不影响请求)
list.requestType string 请求类型
list.name string "api"的字段名称,内容的多少取决于名称的定义,即使其他参数不定义
list.value string 对应名称的值
list.requiredName string 必填名称,根据名称来显示必填或非填提示
list.desc string 对应名称的值的描述
list.uploadfile object 上传文件对象{name:'',value:''},它与list.name,list.value一样.但只属于uploadfile

跨域

如果调试接口时是跨域,那么请调整服务器支持跨域.否则,无法返回正确.
